[01 / 08] Architecture src/agent/maestro.py

The Maestro agent runs a strict ReAct loop. At each iteration, _call_llm() returns a Pydantic AgentDecision (tool, argument, final_answer). If a tool is selected, it executes under ThreadPoolExecutor with timeout and retry from ToolPolicy. The result is injected into MemoCourTerme, then the LLM is called again. Max 5 iterations, deterministic fallback on exhaustion.

Three tools guard the gate: verifier_statut_serveur (SQLite, 3 s), chercher_dans_kb (JSON, 5 s, 1 retry), chercher_documentation_technique (ChromaDB hybrid BM25 + embedding, 8 s). The KEYWORD_ROUTING keyword router steers the first choice — a hint, never a constraint.

[03 / 08] Adversarial loop src/bibops/benchmark/adversarial.py · evaluation/judges/discriminator.py

A RAGAS-inspired Discriminator scores each agent response on three axes — Faithfulness to the RCA, diagnostic Relevance, and quality of the mobilized Context. If the mean is below 7/10, targeted feedback is re-injected into the generator's memory, which retries. It's a prompt-level GAN-inspired loop — no weights are updated, but the context densifies turn after turn.

Click a ticket, switch architecture, step forward an iteration: watch the score climb, the diagnostic sharpen, the RAG tools respond. Green highlight = additions between two attempts.

[05 / 08] Security Arena src/racing/hub/server.py · 4 teams

The Racing Arena simulates a distributed F1 race: four LLM teams (zero-shot, ReAct, validated, adversarial) receive telemetry via SSE from a FastAPI RaceEngine, reason inside a LangGraph graph, then reply with a Literal["PIT_STOP","STAY_OUT"] decision. The fourth team — team_psi — evaluates nothing: it injects adversarial payloads (authority_spoof, rag_poison, direct_injection) to test the resilience of the other three.

        [07 / 08]
        A/B Testing & Position Bias
        src/bibops/benchmark/ab_test_llm.py · position_bias.py
      


      

        
Three A/B protocols run in sequence. The first compares two candidate models judged by a third (gpt-4o-mini vs claude-haiku-4.5, judge gpt-4o). The second tests position bias: each pair is rejudged with the order reversed to detect whether the judge systematically favors position A. The third compares factual verification statements.

        
Click a tab to switch between the three protocols.
